The frameworks
under the hood.

PROMPT ENGINEERING
ROLE × CONTEXT × TASK × OUTPUT
Every great prompt has four layers. Most people only write the Task. The agent enforces all four — and adds constraints to prevent the most common failure modes.
"A prompt is a product spec for a one-shot software system."
AGENT DESIGN
PERCEIVE → REASON → ACT → OBSERVE
The agent loop that actually holds up in production. Design each step with its own failure handling, minimum permissions, and explicit exit conditions.
"Agents fail at boundaries. Design for the seams."
PRODUCT THINKING
THE DRIP FRAMEWORK
Delightful, Remarkable, Insightful, Personalized. AI-native products rethink the workflow entirely — they don't bolt AI onto existing UX patterns.
"The best software doesn't feel like software."
EVALUATION
BUILD EVALS BEFORE FEATURES
Measure accuracy, relevance, tone-fit, instruction-following, and graceful failure. Test edge cases, not happy paths. Real users are creative in their dysfunction.
"You can't improve what you don't measure."
